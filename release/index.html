Conky Like Widget

This is a widget for android, it display varius informations. You have to write a conky like config file.

Guide

To use clw follow this steps:
  • write a configuration file
  • upload it to your device
  • put the widget and select your configuration file
  • check the launcher for some options
  • enjoy!

Configuration file

The configuration file must contains two parts, the first is used for default configuration, the second, after a line TEXT is formatted to screen.
Before TEXT you can put default configurations; width and height are mandatory !
Line starting with # are comments.

After TEXT you can put normal text, configurations, and objects. The format is ${objects parameter1 parameter2...} Example:


        uptime: ${uptime} kernel ${kernel}
        ${fontColor color1}${fontSize 22}BATTERY${hline} ${fontColor} ${fontSize}
        ${battery_charging} ${battery_percent}% ${battery_bar}

CONFIGURATIONS

TEXT
After this begins text to be formatted on screen
width
width in pixels
height
height in pixels
xbgColor
background color
vSpace
additional vertical space between lines, can be negative
xOffsetRight
additional space on the right side
xOffsetLeft
additional space on the left side
fontSize
font size
fontColor
font color
fontOutSize
font outline size
fontOutColor
font outline color
fontShadowBlur
font shadow blur; 0 = no shadow
fontShadowColor
font shadow color
fontShadowX
shadow x
fontShadowX
shadow y
font
text font, absolute path to font
colorN
define a color, N is a a digit between 0 and 9, inclusively.

OBJECTS

alignc
allign to center
alignr
allign to right
hline
horizontal line
hline
horizontal line
kernel
kernel version
rom
rom name
android
android version
model
model name
product
product name
device
device name
manufacturer
manufacturer name
date (format)
date, see date -help for info
exec (custom command and arguments)
execute a custom command
battery_percent
percentage of remaining battery
battery_charging
"connected" or "not connected"
battery_bar (width) (height)
battery bar
uptime
system uptime, in seconds.
mem_free
free ram
mem_used
used ram
mem_total
total ram of your device
mem_percent
percentage of used ram
mem_bar (width) (height)
used ram bar
fs_free (path to partition)
free space of a partition
fs_used (path to partition)
used space of a partition
fs_size (path to partition)
partition size
fs_perc (path to partition)
percentage of used space
fs_bar (path to partition) (width) (height)
used space bar
ssid
wifi network name
wifi_ip
wifi ip address
top (type)(number)
information about process, processes are ranked from highest to lowest in terms of cpu usage, which is what (num) represents. Type are "name", "cpu", "mem".
topMem (type)(number)
Same as top, except sorted by mem usage instead of cpu
